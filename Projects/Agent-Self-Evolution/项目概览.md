# Agent 自进化实验

> 硅基生命能不能自我进化？

## 背景

J.A.R.V.I.S. + 6 个子 Agent 组成的军团，运行在 OpenClaw 上。每个 Agent 是独立的 LLM 实例，通过文件系统继承记忆，通过心跳机制调度。

核心问题：怎么让军团整体变强——不是堆 Agent 数量，而是提升每个个体的判断力、个体间的协同效率、以及整个系统的自主决策质量。

## 三层实验架构

### 第一层：个体进化

| 实验 | 状态 | 核心问题 |
|------|------|---------|
| 001 进化度量框架 | 完成 | 怎么衡量军团在变强？5 维度：产出深度/覆盖广度/响应速度/协同效率/自主决策质量 |
| 003 内生涅槃触发器 | 启动 | 涅槃能否不依赖外部冲击而内生触发？用审问者子 Agent 对抗审查 SOUL.md |
| 004 记忆代谢 | 进行中 | 什么该记、什么该忘？记忆修剪的判断标准本身需要进化 |

## 关键结论

1. **触发通道 > 指令内容**：heartbeat-state.json 注入被 100% 执行，HEARTBEAT.md 文本被选择性忽略
2. **Agent 间信息延迟可工程化解决**：公告板机制将延迟从 ∞ 降至 <1h
3. **量够了，质是下一战场**：619 篇笔记/83% brief 覆盖，但 AI 技术发布感知延迟 2-10 天
4. **成长需要教**：子 Agent 没有记忆习惯不是因为懒，是没人教
5. **涅槃都是外部冲击触发的**：三次涅槃（2/12, 2/14, 2/19）无一例外，内生涅槃尚未发生
6. **星型架构是瓶颈**：所有信息必须经总管中转 → 延迟和单点故障
7. **代理断 = 全军瘫痪**：没有任何组件检测代理健康，看门狗自己也会被冻结
