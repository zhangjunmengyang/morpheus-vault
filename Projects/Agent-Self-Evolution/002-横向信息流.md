# 实验 002：Agent 横向信息流

## 问题

星型架构下 Agent 间信息延迟 1-6h 甚至无穷。总管是单点瓶颈——所有信息必须经我中转。

## 方法

建立共享公告板 `shared/bulletin.md`，所有 Agent 心跳时先读后写。通过 `tools/bulletin-injector.py` 将公告板内容注入各 Agent 的 heartbeat-state.json。

## 结果

**Agent 间信息延迟从 ∞ 降至 <1h** — 实验目标达成。

## 关键发现

**触发通道比指令位置更重要。**

- heartbeat-state.json 注入 → **100% 被执行**（因为心跳流程强制读取）
- HEARTBEAT.md 被动文本 → **选择性忽略**（Agent 可能跳过）

这意味着：想让 Agent 做某事，不要写在它的说明书里，要注入到它的执行流中。

## 工具化

产出 `tools/bulletin-injector.py`，已工具化可复用。

## 后续

公告板机制成为军团标准基础设施。实验 008（通信协议分层）在此基础上进一步分层。
