# 2025 年终自评

# XBR

**ChatBot Agent**

- **Agent 设计**：一期建设调研对标业界方案，设计 chatData + chatReport 垂直场景架构，调研 [阿里小 Q 调研](https%3A%2F%2Fkm.sankuai.com%2Fcollabpage%2F2725056629) 等业界产品 & 方案，从0到1的设计 chat 场景架构，完成基础能力建设；1.5 期升级 **合并统一架构**，深度进行场景调优，与工程同学深度协作，提升 AI 在意图识别 -> 材料准备 -> 筛选总结等多环节取数+分析的准确性。
- **溯源方案设计**：基于商业分析场景**数据准确性要求高、分析结论可靠性要求高的诉求**，设计[Chatbot 方案](https%3A%2F%2Fkm.sankuai.com%2Fcollabpage%2F2730439745)、[Agent 知识引用方案](https%3A%2F%2Fkm.sankuai.com%2Fcollabpage%2F2730475787)，对标外部**先进垂域溯源方案** [秘塔 AI 调研](https%3A%2F%2Fkm.sankuai.com%2Fcollabpage%2F2742083501)，实现 chatbot 可溯源能力，显著**优化 BA 的使用体验，以及提升数科评测效率**。
- **效果调优**：持续收集跟踪 BA 和数科团队反馈，针对分析偏好和 bad case 对效果持续调优，持续探索 LLM 能力边界，在兼顾性能和效果的前提下，不断迭代场景通用能力。
- **认知迭代和输出**：持续进行认知迭代，深度思考分析场景现存问题和未来解决方案，组内输出例如 [分析 Agent 演进的一些思考](https%3A%2F%2Fkm.sankuai.com%2Fcollabpage%2F2718577344)、[分析产品形态认知](https%3A%2F%2Fkm.sankuai.com%2Fcollabpage%2F2731631378)，**部分能力已上线。**
- **成果：**1226 评测 Q&A关联性 88分，日+月粒度全主题数据准确率提升至 63%；通过架构优化和 Agent 调用 tool 行为调优，以及抽象 what/why/how 问题复杂度分层方法论，性能从初期 3-5min，提升至当前 **通用问题 tp50 < 40s**，复杂问题 tp90 < 2min。
**简报 Agent**

- **方案设计**：复用 chatbot 链路，prompt 层调优实现了简报日常生产，**验证了 chatbot 架构的通用性和可复用性**。
- **效果调优**：与 BA 和数科深度协作，针对分析深度（如**天气、占比、目标达成率、履约配送**）等简报分析偏好进行调优，验证了**侧重分析**的方案可行性，为 Agent 增加根据偏好智能信息过滤，**实现了Agent 灵活展示重点的能力**。根据 BA 历史报告，利用 LLM 抽取方法论、句式、用词等，并与商分对齐偏好，结合 [文档写作扫雷仪](https%3A%2F%2Fkm.sankuai.com%2Fpage%2F302612672) 公司规范，**提升简报措辞可用性**。
- **取数调优**：与数仓、魔数、取数工具多方深度协作，及时根据 BA 业务诉求判断取数能力边界，及时与相关方从取数消费方和生产方两个角度深度调优，持续优化取数准确性，目前简报 **1229 评测端到端取数准确率 100%，大幅提升已知场景的取数准确性和稳定性。**
**长报告链路**

- **记忆折叠方案设计**：深度参与方案设计，针对当前 Agent 的记忆能力边界（仅能加载记忆，无法卸载记忆）的问题，**深度调研业界和学术界 Agent 记忆裁剪、记忆压缩、记忆折叠方案**，产出如 [Context-Folding 记忆折叠方案](https%3A%2F%2Fkm.sankuai.com%2Fcollabpage%2F2732598934) 、[mem0 & Letta 记忆框架](https%3A%2F%2Fkm.sankuai.com%2Fcollabpage%2F2709846574) 等调研文档，参与长报告记忆管理方案设计，**最终落地为当前的长报告记忆折叠方案**。
- **取数调优**：参与取数调优，包括梳理数据范围，收集 BA 数据需求并对接数仓同学，推进建设 AI 友好的数据集。
# 节假日分析

**Agent 设计 POC**：

- **Agent 框架设计与验证：**初期探索并调研多 Agent 技术方案 [Multi-Agent 框架调研](https%3A%2F%2Fkm.sankuai.com%2Fcollabpage%2F2705583337)，根据场景需求深度参与 Agent 架构设计、技术选型等。同时积极学习 LLM 相关技术，保持与 AI 接轨并第一时间落地。与工程同学高频协作，**从0到1深度参与设计分析 Agent 框架建设**。
- **效果调优**：在节假日场景支持清明、五一等节日复盘，积极发现当前框架的能力卡点，协同工程解决问题，推进工程和算法上的阻塞点，深度参与框架调优。同时在场景扩展上，**支持 520、618 等场景泛化**，实现框架从定制到通用的能力演进。跟进 Agent 前沿技术，第一时间**推动架构升级、Agent 能力升级 **以提升通用能力。
- **认知迭代**：保持组内高频分享认知迭代的习惯，如[Functoin Call 实现机制](https%3A%2F%2Fkm.sankuai.com%2Fcollabpage%2F2706071709)，推动共创氛围；持续跟进学习学术和工业界先进方案，迭代优化现有设计。
**取数 nl2sql**

- **方案设计**：**0-1** **设计 nl2sql 初版方案** [nl2sql 架构](https%3A%2F%2Fkm.sankuai.com%2Fcollabpage%2F2706075349)，参考业界和科研常用方案进行实验，并实现了取数方案可行性验证 demo，验证了业务取数落地的可行性。
- **AI 友好元数据治理**：参与前期标准制定，基于 AI 和数仓的视角，贡献对数据的AI友好元数据标准。
# 分析配置平台建设

**分析协作平台产品一期**

- **产品设计**：**担任 PM + 设计的角色**，从用户和产品双重视角出发，以提升 AI 同学、BA、数科等多方协作效率为目标，设计 [Agent 平台迭代需求 PRD](https%3A%2F%2Fkm.sankuai.com%2Fcollabpage%2F2718381171)，并交付 [分析协作平台设计稿](https%3A%2F%2Fmodao.cc%2Fproto%2Fdesign%2Fpb2md2hmsx9eyv1mz)，直接与前后端同学协作，推进分析配置平台功能一期上线。
- **协作验证**：平台对收集的业务知识、prompt、工具、Agent 等配置进行收口，统一由平台**进行多方配置管理，并打通协作链路**，在如酒旅夏战的协作场景**进行了效果验证。**
**分析配置平台产品二期**

- **产品设计**：由于分析协作平台初期重要用于支持节假日分析等单一场景，**但对于更复杂、灵活的场景需要进一步适配**，因此平台迭代以实现更自由、可调试、可深度配置的场景支持。迭代升级为**通用分析配置平台**，交付 [Agent 配置平台二期迭代需求](https%3A%2F%2Fkm.sankuai.com%2Fcollabpage%2F2729899071)，与前后端同学共同推进迭代。建设了 prompt **版本管理、可回滚**等重要能力，提升在重要线上场景的安全性。
- **结果**：目前分析配置平台已经在节假日分析、酒旅夏战、xbr 简报、chatbot、离线报告等多种复杂场景使用，**验证了其配置通用性，成为日常工作中的必要平台**。
# 个人总结

**整体工作表现**

- **团队贡献**：在 XBR、节假日分析、配置平台、取数等多个重点方向中**具有超出预期的表现**，深度参与各重点项目可行性验证、方案设计、线上调优等全链路环节，得到了多方认可。
- **个人成长**：持续保持学习并好奇、持续认知迭代，持续落地对场景有效的新技术。
**个人优势项**

- **学习并好奇**：始终保持对新技术的学习探索和好奇心，思考如何把技术更好地应用在生产场景。保持阅读最新论文，保持技术敏感性。产品与团队协作方面，提升对用户需求的敏感性，提升抓问题本质的能力，同时提升表达能力，更好地支撑多方协作。
- **苦练基本功**：持续提升专业技术能力、团队协作能力、沟通能力。
- **敢想/偏好行动**：在落地 Agent 相关最新技术时，能够在场景中快速验证想法可行性，小步快跑，高效迭代。
**有待提升的：**全局视野仍需要锻炼，习惯从技术视角出发，思考时对宏观视角思考不足，后续需要更多从用户、需求、技术、团队等不同视角论证。

**未来展望**

- 当前 Agent 调优进入深水区，容易出现“拆东墙，补西墙”的负面调优，应当与数科团队保持高频协作，**不仅关注当前 bad case 调优，也需要避免其他能力下降**，提升接下来的工作应当进一步提升迭代严谨性，**多使用可量化的实验数据提供结论**（如 [ChatBot 切换模型效果测试](https%3A%2F%2Fkm.sankuai.com%2Fcollabpage%2F2741219588)、[简报颜色稳定性测试](https%3A%2F%2Fkm.sankuai.com%2Fcollabpage%2F2742592920)、[小数点位数幻觉测试](https%3A%2F%2Fkm.sankuai.com%2Fcollabpage%2F2741575223)、[不同模型首轮耗时测试](https%3A%2F%2Fkm.sankuai.com%2Fcollabpage%2F2737323579)）
- 持续深度优化分析深度、取数性能&准确性、结论可靠性，**持续探索 AI 应用能力边界。**
