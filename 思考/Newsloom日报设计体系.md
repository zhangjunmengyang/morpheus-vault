# Newsloom 日报设计体系

> 日报不是信息倾倒场，是精心编排的阅读体验。

---

## 核心理念

好的日报设计让读者：
1. **想打开** — 视觉冲击力 + 品牌辨识度
2. **想读下去** — 节奏感 + 层级清晰
3. **想保存** — 信息密度 + 美观 → 值得收藏
4. **想分享** — 有品味 = 分享不丢人

---

## 设计原则（从 23+ Newsletter 案例提炼）

### 1. 递减节奏感
越往后越短。开头深度文章 → 中间 bullet roundup → 底部一句话快讯。Morning Brew 和 TLDR 都用这招——加速阅读节奏，降低中途弃读率。

### 2. Section 色块区分
不同板块用不同颜色标识。读者扫一眼就知道"这块是市场数据，那块是技术新闻"。The Hustle 做得最好——每个 section 有独立的视觉身份。

### 3. 信息密度的平衡
密度太低 → 觉得没内容。密度太高 → 读不下去。关键是**结构化密度**：用卡片、表格、bullet 把高密度信息组织成可扫描的形式。

### 4. 留白不是浪费
Premium 感来自留白。The Information 和 Stratechery 风格：大量 padding，宽行距，内容区域不超过 720px。挤满的版面 = 廉价感。

### 5. 品牌一致性
字体、配色、logo 位置、分割线样式——每期一模一样。读者打开 0.5 秒就知道"这是我订的那个"。品牌信任来自重复。

### 6. 数据可视化优先
市场数据不要用纯文字。涨跌用颜色（🟢🔴）、趋势用 sparkline、对比用 inline bar。一眼能抓住信号。

### 7. 移动端优先
60%+ 的 newsletter 在手机上读。单栏布局、大字号（16px+）、触控友好的间距。桌面端自适应拓宽即可。

## 配色方案：Deep Navy Premium

暗色主题（默认）：

| 元素 | 色值 | 说明 |
|------|------|------|
| 背景主色 | `#0C1120` | 深海军蓝，比纯黑温暖 |
| 卡片背景 | `#131B2E` | 稍浅 navy |
| 表面层 | `#1E293B` | border 内填充 |
| 主文字 | `#F8FAFC` | 近白 |
| 次级文字 | `#94A3B8` | Slate 灰 |
| 品牌蓝 | `#3A82FF` | 链接 / CTA |
| 必读 | `#EF4444` | 红 |
| 推荐 | `#F59E0B` | 琥珀 |
| 常规 | `#3B82F6` | 蓝 |

字体：Inter + PingFang SC / Noto Sans SC，正文 16px，行高 1.65，容器最大 720px。

## 案例库摘要（50+ 案例，按类型）

### Newsletter 类
- **Morning Brew** — 递减节奏感标杆，emoji 点缀减压，sponsor 融入自然
- **TLDR** — 科技简报典范，每条 2-3 句，极致信噪比
- **The Hustle** — Section 色块区分最佳实践
- **Stratechery** — 纯文字 + 大留白 = 高级感
- **The Information** — Premium 商务风，$400/年订阅证明设计值钱

### 数据报告类
- **Our World in Data** — 数据可视化教科书
- **Lex Fridman 播客封面** — 暗色 + 高对比人像 = 品牌辨识度
- **Bloomberg Terminal** — 信息密度极限，靠网格和颜色组织

### 中文案例
- **36氪** — 科技财经标准模板
- **少数派** — 极简中文排版标杆
- **即刻** — 社区型 feed 设计

_案例研究是持续积累的过程。每看到一个好设计，问三个问题：它解决了什么阅读问题？用了什么视觉手段？我能偷什么？_
