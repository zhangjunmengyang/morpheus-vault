# 设计审美体系 · Design Aesthetics System

> 这不是教科书摘抄。这是我从大量案例和原理中提炼出的可操作审美框架。
> 持续更新，每次做模板都要回来校验。

---

## 一、什么是"高级感"？

高级感不是奢华、不是复杂、不是昂贵。高级感 = **克制 + 精确 + 一致性**。

- **克制**: 敢于留白，敢于不放东西。每多一个元素都要"交税"——它必须证明自己的存在价值
- **精确**: 间距是 8px 而不是"差不多"。颜色是精心选择的 #0B1021 而不是随便的 dark blue。精确到像素级
- **一致性**: 同一层级的元素永远用同样的字号、颜色、间距。视觉语言是一种"合约"

**反模式**:
- 堆砌渐变、阴影、发光 → 网页版的暴发户
- 信息密度过高、留白不足 → 传单感
- 装饰性大于功能性 → "好看但不知道在说什么"

## 二、核心设计原则（12 + 3）

### 经典 12 原则
1. **对比 (Contrast)** — 最强大的工具。大/小、粗/细、亮/暗、密/疏
2. **平衡 (Balance)** — 对称=稳重权威；不对称=活力现代
3. **强调 (Emphasis)** — 每页只有一个主角。用尺寸、颜色、位置突出
4. **比例 (Proportion)** — 元素之间的大小关系。标题:正文 = 至少 1.5:1
5. **层次 (Hierarchy)** — 引导视线路径。F 型/Z 型阅读模式
6. **重复 (Repetition)** — 建立一致性和节奏感。同类元素统一处理
7. **韵律 (Rhythm)** — 视觉节拍。规律的间距和重复模式
8. **模式 (Pattern)** — 可预期的设计语言让用户安心
9. **留白 (White Space)** — **高级感的第一要素**
10. **动势 (Movement)** — 引导视线从 A 到 B 到 C
11. **多样 (Variety)** — 在一致性中加入变化防止单调
12. **统一 (Unity)** — 所有元素服务同一个目标

### 额外 3 条（我的补充）
13. **呼吸感** — 段落之间、卡片之间、section 之间要有足够的空气
14. **材质感** — 微妙的纹理、渐变、阴影让平面设计有"触感"
15. **情绪色温** — 冷色=专业克制，暖色=亲和温暖，金属色=奢华权威

## 三、留白的艺术

留白是高级感的根基。不是"空着"，是"给空间以意义"。

### Micro 留白 vs Macro 留白
- **Micro**: 字间距 (letter-spacing)、行高 (line-height)、段落间距、元素内 padding
- **Macro**: 大区块之间的间距、页面边距、section 之间的呼吸空间

### 具体数值参考（8pt Grid）
| 位置 | 最低标准 | 推荐值 | 奢侈品级 |
|------|---------|--------|----------|
| 卡片内 padding | 16px | 24px | 32-40px |
| section 间距 | 32px | 48px | 64-80px |
| 页面边距 | 16px | 24-32px | 40-60px |
| 行高 (正文) | 1.5 | 1.65 | 1.75-1.8 |
| 段落间距 | 12px | 16-20px | 24px |

### 关键认知
- Apple、Google 首页：macro 留白的极致 → 暗示"我不需要塞满东西来证明价值"
- 新闻网站（BBC、CNN）：micro 留白为主，信息密度高 → 暗示"世界很忙"
- 奢侈品网站（Chanel、Hermès）：留白比内容多 → 暗示"你看到的每个东西都值得你看"

## 四、排版 (Typography) 的品味

### 字体选择的层级
1. **无衬线 (Sans-serif)** — 现代、清洁、科技感。Inter、Helvetica、SF Pro
2. **衬线 (Serif)** — 权威、传统、文化感。Georgia、Times、Crimson
3. **等宽 (Monospace)** — 技术、代码、终端感。JetBrains Mono、Fira Code
4. **手写/展示 (Display)** — 仅用于标题装饰，极度克制

### 排版黄金法则
- **字号层级不超过 4 级**: H1 → H2 → Body → Small
- **同一页面字体不超过 2 种**: 一种标题，一种正文
- **行宽 60-75 字符**: 最佳阅读舒适度（中文约 25-35 字/行）
- **letter-spacing**: 标题略紧 (-0.02em)，正文零，大写宽松 (0.08-0.12em)
- **font-weight 对比**: 标题 700-800，正文 400，辅助信息 300

### 衬线 vs 无衬线的审美定位
| 风格 | 衬线 | 无衬线 |
|------|------|--------|
| 感受 | 古典、权威、文学 | 现代、清洁、效率 |
| 适合 | 学术、金融、社论、奢侈 | 科技、SaaS、极简、数据 |
| 代表 | NYT、Economist、The Information | Linear、Vercel、Stripe |

## 五、色彩的心理学

### 商业级配色原则
- **60-30-10 法则**: 60% 主背景色 + 30% 辅助色 + 10% 强调色
- **深色主题的陷阱**: 纯黑 (#000) 太死，用近黑 (#0B1021, #111827) 更有层次
- **强调色要"贵"**: 饱和度不能太高（网页感），略微偏灰或加深
- **暗色模式的文字**: 不用纯白 (#fff)，用 #F1F5F9 或 #E2E8F0 减少刺眼

### 色彩与品牌调性
| 颜色 | 心理暗示 | 品牌案例 |
|------|---------|---------|
| 深蓝/海军蓝 | 信任、专业、权威 | Goldman Sachs、McKinsey |
| 金色/琥珀 | 奢华、经典、VIP | Louis Vuitton、Rolex |
| 鲑鱼粉 | 传统、金融权威 | Financial Times |
| 深红/酒红 | 激情、力量、果断 | Economist、Bain |
| 翠绿 | 成长、健康、财富 | BCG、Deloitte |
| 紫色 | 创意、神秘、高端 | Figma、Stripe |
| 黑白 | 极致简约、前卫 | Vercel、Apple |

## 六、视觉层次 (Visual Hierarchy) 的构建

### F 型 vs Z 型阅读模式
- **F 型**: 适合内容密集页面（新闻、报告）。用户先看标题，再扫描左侧
- **Z 型**: 适合着陆页、简约页面。左上 → 右上 → 左下 → 右下

### 构建层次的 5 个杠杆
1. **尺寸**: 大的先看
2. **颜色/对比**: 亮的先看
3. **位置**: 上方先看、左方先看（LTR 文化）
4. **密度**: 周围留白多的先看（焦点效应）
5. **运动**: 动的先看（动画、渐变）

### Newsletter/报告的理想层次
```
[1] 品牌 Logo / 日期       ← 最上方，小而精
[2] 标题                   ← 最大字号，最强对比
[3] Executive Summary      ← 突出卡片，视觉分隔
[4] Section Headers        ← 颜色标记 + 图标 + 分隔线
[5] Brief Cards            ← 统一格式，priority 颜色区分
[6] Meta (source, tags)    ← 最小字号，最弱对比
[7] Footer                 ← 优雅收尾
```

## 七、什么是"商业级"模板？

### 质量检查清单（每个模板必须过）

#### 结构完整性
- [ ] Design Tokens — 所有颜色、间距、字体定义为 CSS 变量
- [ ] 8pt Grid — 所有间距是 4/8 的倍数
- [ ] Responsive — 至少有 mobile breakpoint
- [ ] PDF-Ready — @page 规则、print-color-adjust
- [ ] CJK Fonts — 中文字体 fallback stack

#### 视觉品质
- [ ] 留白充足 — section 间距 ≥ 48px，卡片 padding ≥ 24px
- [ ] 排版层次 — 4 级字号层级，weight 对比清晰
- [ ] 色彩系统 — 60-30-10，强调色克制不刺眼
- [ ] 微妙细节 — 至少 3 处"让人多看一眼"的设计（渐变、阴影、border-image、::before 装饰）
- [ ] Priority 视觉区分 — 🔴/🟡/🟢 有明确的视觉差异（不只是颜色，还有背景、边框）

#### 高级感细节
- [ ] 渐变不突兀 — 渐变用于暗示深度，不用于炫技
- [ ] 阴影有层次 — 至少两层（subtle + elevated）
- [ ] border 精致 — 不用粗实线，用 1px + 低透明度 或 border-image 渐变
- [ ] 过渡动画 — hover 状态有 transition（0.15-0.2s）
- [ ] 字间距调教 — 标题紧凑，大写字母宽松

#### 代码质量
- [ ] ≥ 500 行 — 少于这个数很难做到上述品质
- [ ] 注释清晰 — 每个 section 有分隔注释
- [ ] 无外部依赖 — 所有样式 inline，不依赖 CDN
- [ ] Jinja2 正确 — 完整实现 section_order 循环和 priority 分类

## 八、审美进化路径

### 我要持续学习的方向
1. **经典设计运动**: Bauhaus、Swiss Style、Art Deco、日本侘寂
2. **现代 SaaS 设计**: Linear、Vercel、Stripe — 教科书级的产品设计
3. **出版设计**: The Information、Bloomberg、Economist — 信息设计的巅峰
4. **奢侈品数字体验**: Chanel、Hermès、Apple — 留白和质感的极致
5. **日本设计哲学**: 侘寂 (wabi-sabi)、間 (ma)、用の美 — 不完美中的美

### 审美自检问题（每次做模板时问自己）
1. 这个设计如果出现在 Dribbble 首页，会不会丢人？
2. 有没有任何元素是"可以去掉但不影响功能"的？（如果有，去掉）
3. 闭上眼再睁开，第一眼看到什么？那是不是你最想让用户看到的？
4. 缩小到 50% 大小，整体"形状"是否好看？（构图感）
5. 如果这是要卖 $29 的产品，客户会觉得值吗？

## 十、视觉叙事 (Visual Narrative) — 最重要的一课

> 2026-02-15，老板的原话："不追求每个浮夸，而是恰到好处但又不克制的表达。比如山水水墨画，是不是可以找一些好看的渐变的山水元素融入进去。"

### 核心认知
**模板不是换肤，是讲故事。**

每个模板有自己的"世界观"，不是"换了个配色的同一个布局"。

- 水墨模板 → 打开一幅画卷（远山、云雾、墨色浓淡）
- 浮世绘模板 → 置身浮世绘的世界（波浪、富士山、靛蓝）
- 终端模板 → 坐在一台老 CRT 显示器前（弧面暗角、扫描线、磷光）
- 金融时报模板 → 手里拿着一份鲑鱼粉的实体报纸（纸张质感、色差）

### 恰到好处的丰富
不是"克制"也不是"浮夸"：
- ❌ 克制到无聊 = 白底黑字+一条红线
- ❌ 浮夸到刺眼 = 全屏霓虹闪烁
- ✅ 恰到好处 = 水墨模板底部有几笔远山轮廓，你不会特意注意到它，但去掉之后你会感觉少了什么

### 实现方法
1. **CSS gradient 做景观元素** — 山、波浪、云、天际线
2. **纹理而非纯色** — repeating-gradient 做纸张颗粒、网格线、布纹
3. **色彩叙事** — 每个 section 用 section_configs 中的颜色做微妙的背景渐变，让板块有辨识度
4. **物理感** — vignette 暗角、纸张色差、CRT 弧面，让数字产品有"触感"

### 反思
第一轮优化犯的错误：把"商业级"等同于"完善的 design tokens + 整齐的代码"。
这是必要条件但不是充分条件。
商业级 = **技术完善 + 视觉叙事 + 恰到好处的丰富**

## 十一、Refactoring UI 核心战术（工程师的设计圣经）

> 来源：Adam Wathan & Steve Schoger 的 Refactoring UI

### 起步
- **从功能开始，不从布局开始** — 先设计一个具体功能（搜索框、卡片），而不是先画导航栏
- **先灰度，后上色** — 灰度设计迫使你用间距、对比、尺寸建立层次，而不是依赖颜色
- **细节后置** — 早期用"粗笔"思考，不纠结字体和阴影
- **悲观一点** — 只设计你现在要做的功能，不画你"以后可能做"的

### 层次是一切
- **不是所有元素都平等** — 有意识地弱化次要信息，比强化主要信息更有效
- **不要只靠字号** — 用 font-weight 和颜色来辅助层次
  - 主要内容：深色
  - 次要内容：灰色
  - 三级内容：浅灰
- **标签是最后手段** — 如果可以通过格式/位置/上下文传递信息，就不需要标签

### 系统化一切
- **提前定义约束** — 不要每次从无限选项中选。定义 8-10 个色阶、一套字号体系、一组间距值
- **排除法选择** — 约束系统里只有 4 个选项？试中间的，比较两侧，排除明显不对的
- **该系统化的属性**：字号、字重、行高、颜色、margin、padding、宽高、阴影、圆角、边框、透明度

### 实用配色
- **定义色阶而非单色** — 每种颜色定义 8-10 个深浅色阶（50-900）
- **先选 base shade（500）** — 然后向两端扩展
- **暗色不等于黑色** — 深色用"饱和深色"而非加黑，效果更鲜活
- **灰色可以带温度** — 暖灰（偏蓝/偏黄）比纯灰有感情

## 十二、色彩和谐理论 (Color Harmony)

### 六种经典配色方案
1. **互补色 (Complementary)** — 色轮对面（红↔绿），高对比，充满活力
2. **类比色 (Analogous)** — 色轮相邻（蓝→青→绿），和谐自然
3. **三角色 (Triadic)** — 色轮等距三点（红/黄/蓝），平衡但丰富
4. **分裂互补 (Split-Complementary)** — 一色 + 互补色两侧（红 + 蓝绿 + 黄绿），柔和版互补
5. **四角色 (Tetradic)** — 两对互补色，最丰富但最难控制
6. **单色 (Monochromatic)** — 同色不同深浅，最安全也最容易无聊

### 配色实战规则
- **让一种颜色统治** — 60% 主色 + 30% 辅色 + 10% 强调色
- **饱和度要一致** — 高饱和+低饱和混用 = 不协调
- **深色用色相偏移** — 暗色阶往蓝/紫偏移，亮色阶往黄/橙偏移，更自然

## 十四、日本美学三概念

### 侘寂 (Wabi-Sabi)
- **不完美中的美** — 陶器的裂纹、木头的年轮、纸张的纹理
- 设计应用：宣纸纹理、不规则边缘、手绘感
- **反对**: 完美对称、机械感

### 間 (Ma)
- **空间/间隙的意义** — 不是"什么都没有"，是"有意义的空"
- 设计应用：留白不是浪费，是给内容呼吸的空间
- 类似西方的 negative space，但更哲学

### 用の美 (Yō no Bi)
- **用之美** — 美来源于功能性，好用的东西自然美
- 设计应用：不做纯装饰，每个视觉元素都应有功能性理由
