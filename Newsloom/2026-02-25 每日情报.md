**Dev.to AI** #AgentSecurity #MultiAgent

AutoGen/CrewAI/LangGraph 等框架构建的 Agent 链中，Agent A 委托 Agent B 时没有身份验证、消息签名或权限边界——任何能注入中间消息的攻击者都可以伪装成合法 Agent 发指令。所以呢？你在设计 multi-agent 工作流时，必须把 Agent 间通信当成不可信信道来设计，至少加 HMAC 签名或 token 验证，否则 prompt injection 可以横向传播整条 Agent 链。
