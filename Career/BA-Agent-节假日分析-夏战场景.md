# BA Agent 节假日分析 · 夏战场景落地

## 一、背景与目标

节假日场景（第二期）已沉淀了分析 Agent 的产品化能力基础。本阶段目标：

- **场景泛化**：将节假日能力快速复用至夏战场景（酒旅、闪购），边实战边打磨
- **平台产品化**：完善分析 Agent 协作平台，支持商分、数科等角色高效协同
- **降低成本**：解决配置项高度耦合、调试成本高、偏好对齐轮次多等核心痛点

---

## 二、平台产品化迭代

### 2.1 配置项三层解耦

**核心问题**：原配置项高度耦合，对多角色分工协作不友好。

**解决方案**：按角色和协作流程将配置拆分为三层：

| 配置层 | 目标角色 | 主要内容 |
|--------|---------|---------|
| 全局配置 | 平台管理员 | 负责人、描述、知识库、数据集等基础信息 |
| 业务配置 | 商分 | 分析思路、分析框架、补充知识、Excel 自定义数据上传 |
| 高级配置 | 数科/算法 | 全部 Agent prompt、模型参数，供深度调优 |

**产品化价值**：不同角色按权限访问对应层，协作链路清晰，降低单场景调试成本。

### 2.2 人工干预能力

支持三个关键节点的人工修正，保障报告质量：

1. **Planner 规划修正**：Planner 根据业务配置和分析时间自主规划步骤，用户可针对输出做修正，保障规划阶段的正确性
2. **章节表格替换**：受限于取数准确率，支持用户将错误表格内容修正后重新上传，并重新生成当前章节
3. **最终报告标注**：协作平台支持报告标注，推荐 copy 至学城进行深度修改和评论

### 2.3 Agent 调试能力

新增工具调用级别的细粒度日志，为高级用户提供全流程深度调优能力。

### 2.4 Excel 自定义数据分析

支持用户上传 Excel 文件，Agent 在 Plan 阶段根据用户对文件的说明，自动将数据分配至对应章节，实现自定义分析流程。

## 三、场景泛化实战

### 3.1 夏战场景试点结果

在酒旅和闪购两个业务线试点夏战策略复盘场景：

- **酒旅**：业务商分已通过分析协作平台完成分析框架和偏好配置，**已产出业务可用报告并投产** ✅
- **闪购**：初步跑通报告产出流程，待数据集建设完善后继续效果调优 🔄

### 3.2 数据准备现状与演进

**现状**：数据准备工作均为线下完成，周期长、成本高。

**演进路径**：打通魔数体系（起源 2.0 标准指标取数 + 魔数个人数据集取数），用户可从已有的非标+标准数据集取数，基础能力开发已完成。

## 四、评测自动化

### 4.1 结论可靠性评测体系

数科沉淀的评测能力：通过人工评测经验将分析结论错误划分为 **6 类**，并利用 LLM 自动化识别：

**逻辑论证类（3 类）**：
- **推论不合理**：数据与结论间存在逻辑矛盾（最严重）
- **推论不充分**：逻辑正确但数据不足以支撑结论
- **无数据支撑**：完全未提供可支撑结论的数据

**概念表述类（3 类）**：
- **指代不明**：对象不清晰或表意存在歧义
- **无意义对比**：对比口径不一致（如爆发系数 vs. 同比混用）
- **概念混淆**：因概念理解有误导致结论不合理（如高线城市≠头部商家）

### 4.2 两阶段错误修正机制

| 阶段 | 机制 | 作用 |
|------|------|------|
| 事中修正 | 质检-修正模块，逐句校验并识别错误类型，调用修正模块纠正 | 从源头降低错误比例 |
| 事后标识 | 二次检测模块，对已生成内容复核并高亮标记疑似错误点及类型 | 提升报告可用性 |

### 4.3 自动化评测实现方案

- **评审流程**：组长助理预分析识别洞察型句子 → 多 Member 独立评审 → 有分歧句子进入辩论环节（最多 2 轮）→ 组长终审
- **评审颗粒度**：句子级精确评审，区分洞察型句子与陈述型句子，前者才进入评审
- **置信度输出**：高/低两档，低置信度结论交由人工校准
- **评估维度**：分析思路、数据处理、方法应用、逻辑思维（通用评测集约 200 题）

## 五、问题与优化方向

### 5.1 效率问题

| 问题 | 短期方案 | 长期方案 |
|------|---------|---------|
| 数据准备成本高 | 打通魔数个人数据集；优先选数据集完备场景（如 WBR） | 用户圈选指标维度，平台自动生成取数 Prompt |
| 偏好对齐轮次多 | 进一步抽象配置项，高频场景 Prompt 自动生成 | 建设经验记忆系统，从用户交互中持续学习（需 PM 角色配合） |
| 高级配置调试成本高 | Prompt 模板泛化性提升 | 分析场景专属 Agent 模板沉淀 |

### 5.2 质量问题

**分析结论事实性错误**：通过事中修正 + 事后标识两阶段策略处理（见第四节）。

**分析全面性与侧重性平衡**：同时产出精炼版 + 全面版两个报告，使用方优先用精炼版，通过全面版补足缺失信息。

**分析方法与场景适配性不足**：不同分析场景对应的分析方法知识沉淀不足（如酒旅一带二策略复盘应侧重 AB 对比而非常规异动分析）。演进方向：
1. 持续沉淀各场景适配的分析方法，内化为 Agent 应用模板
2. 借助 200 题通用评测集辅助基座模型选型，详见分析 Agent 双轨评测建设

> **待决策**：评测集建设专业性极强，需要更多高阶商分专家参与，以保障质量基线和评测结果可靠性；经验记忆系统高度依赖专业产品交互设计，当前团队缺少 PM 角色。
